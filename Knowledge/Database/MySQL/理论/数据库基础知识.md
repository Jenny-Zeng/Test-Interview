1. 数据库事务及含义
事务是一组操作的集合，是一个不可分割的工作单位，事务会把所有的操作作为一个整体一起向系统提交或撤销操作请求，即这些操作要么同时成功，要么同时失败。
2. 事务的四大特性（ACID）
原子性：事务是不可分割的最小操作单元，要么全部成功，要么全部失败
一致性：事务完成时，必须使所有的数据都保持一致性状态
持久性：事务一旦提交或回滚，它对数据库中的数据的改变就是永久。
隔离性：保证事务在不受外部并发操作影响的独立环境下运行。（A事务操作事务时，不影响B）
3. 如何对查询命令进行优化？
- 原因：

        1. 缺乏合适的索引
        2. 查询语句不优化：查询语句本身不高效，可能包含复杂的连接、子查询或不必要的操作。
        3. 大数据量：当数据量增加到一定程度时，查询性能可能会下降。
        4. 硬件资源不足：数据库服务器的CPU、内存、磁盘等资源不足。
        5. 锁和并发问题：频繁的锁竞争或并发操作导致查询阻塞。
- 解决方法：

        1. 添加索引：为经常用于查询条件的列添加索引，加快查询速度。
        2. 优化查询语句：避免使用 SELECT *、减少子查询、尽量避免使用 OR 操作符等，优化查询语句结构。
        3. 分页查询：对于大数据量，可以分页查询或限制返回的行数。
        4. 数据库优化：定期清理无用数据、重建索引、优化数据库配置参数等。
        5. 使用缓存：对于频繁查询的结果，可以考虑使用缓存来提高查询速度。
        6. 数据库水平分片：对于大数据量，可以考虑将数据库进行水平分片，分散数据存储。
        7. 使用数据库性能工具：如Explain分析、慢查询日志等工具来定位查询慢的原因。
        8. 升级硬件资源：如果硬件资源不足，考虑升级数据库服务器的CPU、内存或磁盘。

4. 数据库的优化
    1. 创建合适的索引，避免过多索引
    2. 查询优化
    3. 根据负载调整最大连接数
    4. 读写分离，分库分表、开销
    5. 连接池：减少连接建立
